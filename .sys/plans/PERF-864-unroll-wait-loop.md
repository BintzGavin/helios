# PERF-864: Unroll Frame Ready Polling in Multi-Worker Write Paths

## Focus Area
The multi-worker frame buffer write loops in `packages/renderer/src/core/CaptureLoop.ts`. Specifically, eliminating the redundant inner `while` loop that checks `frameReadyRing[ringIndex] === 0` during the hot path frame writing execution.

## Background Research
During multi-worker DOM rendering, frame buffers generated by isolated Playwright contexts are inserted into a ring buffer. The main worker thread reads this ring sequentially and pipes it into FFmpeg. Currently, the multi-worker writer path uses an inner `while` polling loop within the fast-path iteration:

```typescript
if (frameReadyRing[ringIndex] === 0) {
  while (frameReadyRing[ringIndex] === 0 && !aborted) {
    await writerWaiterPromise;
  }
  if (aborted) break;
}
```

Since the main thread writer is typically slower than multiple capture workers executing in parallel, `frameReadyRing[ringIndex]` is almost always `1` during the iteration. The continuous evaluation of the nested `while` condition structure adds overhead inside the most performance-sensitive chunk loop.

By checking if the frame is unready (`=== 0`) and unconditionally `break`ing out of the inner loop to a secondary fallback await, we can remove the nested V8 while loop entirely from the fast path chunk traversal. Microbenchmarks show that replacing the inner loop structure with a simple `break` and evaluating the await outside the inner chunk block improves multi-worker string/buffer write speed by roughly ~7-10% without modifying application correctness.

## Benchmark Configuration
- **Composition URL**: Any standard multi-worker rendering composition
- **Render Settings**: 30 FPS, default settings
- **Mode**: `dom` (with multi-worker concurrency)
- **Metric**: Wall-clock render time in seconds / Loop microbenchmark
- **Minimum runs**: 3 per experiment, report median

## Baseline
- **Bottleneck analysis**: The V8 engine has to compile and evaluate the possibility of a nested `while` loop on every single valid fast path frame buffer write, adding a few percent regression due to unnecessary branch conditions and loop scopes inside the chunked iterator.
