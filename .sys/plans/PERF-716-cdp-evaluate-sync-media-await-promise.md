# PERF-716: Omit default CDP evaluate params in syncMedia

## Focus Area
`CdpTimeDriver.ts` - `Runtime.evaluate` calls for syncing media elements via CDP.

## Background Research
Currently, in `CdpTimeDriver.ts`, `this.singleFrameSyncMediaParams` and `this.multiFrameSyncMediaParams` objects have `awaitPromise: false`.
The injected DOM script for `window.__helios_sync_media()` returns the number of media elements. Although it is synchronous, `Runtime.evaluate` has different internal Chromium processing paths depending on whether `awaitPromise` is `true` or `false`.
When `awaitPromise: false`, V8 immediately wraps the return value in a CDP response. But wait, if it's synchronous, why does it matter? Actually, the CDP protocol documentation specifies that `awaitPromise: false` allows immediate return, but some internal IPC messaging in V8 might be faster or slower depending on the flag, especially regarding microtask queues within the browser context. Let's see if forcing V8 to `awaitPromise: true` or completely omitting the `awaitPromise` flag (letting it default to `false`) changes the IPC payload size or processing overhead. Wait, better yet, `awaitPromise` defaults to `false`. Providing it explicitly might be adding unnecessary bytes to the CDP payload being serialized and sent over the WebSocket/IPC connection on *every single frame*.

Actually, looking at `this.singleFrameSyncMediaParams = { expression: "window.__helios_sync_media();", awaitPromise: false, returnByValue: false };`
We can optimize the IPC payload size sent to Playwright and forwarded to Chromium by removing unnecessary keys from these params. `awaitPromise: false` and `returnByValue: false` are defaults in the CDP `Runtime.evaluate` command. Removing them shrinks the JSON payload generated by `this.client!.send('Runtime.evaluate', ...)` on every frame by ~40 characters per frame.
Over a 30fps * 10s = 300 frames, that's minor, but in a hot loop, shorter strings mean less JSON stringify overhead in Node and less parsing overhead in Playwright/Chromium IPC.

## Implementation Spec

### Step 1: Remove default parameters from CDP `Runtime.evaluate` payloads
**File**: `packages/renderer/src/drivers/CdpTimeDriver.ts`
**What to change**:
In `CdpTimeDriver.ts`, modify `singleFrameSyncMediaParams` and `multiFrameSyncMediaParams` to omit `awaitPromise: false` and `returnByValue: false`.

```typescript
// Change this:
private singleFrameSyncMediaParams: any = { expression: "window.__helios_sync_media();", awaitPromise: false, returnByValue: false };

// To this:
private singleFrameSyncMediaParams: any = { expression: "window.__helios_sync_media();" };
```

And inside `defaultSyncMedia` method:
```typescript
// Change this:
              this.multiFrameSyncMediaParams[i] = {
                expression: "window.__helios_sync_media();",
                contextId: this.executionContextIds[i],
                awaitPromise: false,
                returnByValue: false
              };

// To this:
              this.multiFrameSyncMediaParams[i] = {
                expression: "window.__helios_sync_media();",
                contextId: this.executionContextIds[i]
              };
```

**Why**: `awaitPromise: false` and `returnByValue: false` are the default behaviors for CDP's `Runtime.evaluate`. Including them explicitly forces V8 to stringify, transmit, and parse these keys for every frame capture loop. Eliminating them reduces IPC payload size and JSON serialization/deserialization CPU cycles in the hot loop.
**Risk**: If Playwright or the CDP connection requires explicit keys for some undocumented reason, `Runtime.evaluate` might behave unexpectedly, but the CDP spec states they are optional defaults.
