# PERF-031: Pre-compile SeekTimeDriver evaluate script via CDP CallFunctionOn

## Focus Area
The Frame Capture Loop in `packages/renderer/src/drivers/SeekTimeDriver.ts`. Specifically, the time synchronization via `CDPSession`'s `Runtime.evaluate` in the `setTime` method.

## Background Research
In PERF-018, the `SeekTimeDriver` was optimized to pass arguments directly to the evaluate function instead of constructing a new string script for every frame. This eliminated repetitive string serialization and V8 compilation time per frame.
However, in PERF-025, the Playwright `page.evaluate()` abstraction was bypassed in favor of raw CDP commands:
```typescript
        const promise = this.cdpSession.send('Runtime.evaluate', {
          expression: `window.__helios_seek(${timeInSeconds}, ${this.timeout})`,
          awaitPromise: true,
          returnByValue: true
        }).then((response) => {
```
This inadvertently reintroduced the V8 compilation bottleneck, as `expression` is constructed as a dynamic string for every single frame. This forces Chromium's V8 engine to parse and compile the `__helios_seek(...)` JavaScript string on every frame.
Instead of using `Runtime.evaluate` with a dynamic string, we should use `Runtime.callFunctionOn`. This allows us to pass a pre-compiled function declaration and a list of arguments directly, avoiding string recompilation.

## Implementation Spec

### Step 1: Use CDP Runtime.callFunctionOn
**File**: `packages/renderer/src/drivers/SeekTimeDriver.ts`
**What to change**:
In the `setTime` method, change the `CDPSession.send('Runtime.evaluate', ...)` call in the `if (this.cdpSession && frame === page.mainFrame())` block.
Instead of sending a `Runtime.evaluate` command with a dynamically constructed string expression, we need to pass the arguments properly using `Runtime.callFunctionOn`.

First, in the CDP initialization logic (e.g., in `prepare` or wherever `cdpSession` is initialized), evaluate a simple script via CDP to get the `objectId` of the `window` object so it can be used for `Runtime.callFunctionOn`. Alternatively, evaluate `this` to get the global execution context.
Then, in `setTime`, use:
```typescript
        const promise = this.cdpSession.send('Runtime.callFunctionOn', {
          objectId: windowObjectId, // The object ID retrieved during initialization
          functionDeclaration: 'function(t, timeoutMs) { return window.__helios_seek(t, timeoutMs); }',
          arguments: [{ value: timeInSeconds }, { value: this.timeout }],
          awaitPromise: true,
          returnByValue: true
        }).then((response) => {
          if (response.exceptionDetails) {
            throw new Error(`Seek error in main frame: ${response.exceptionDetails.exception?.description || 'Unknown error'}`);
          }
        });
```

**Why**: Passing a small function declaration and an argument object allows V8 to compile the wrapper function once or efficiently pass arguments, bypassing the dynamic string parsing overhead that was reintroduced in PERF-025.
**Risk**: The CDP syntax for `callFunctionOn` requires an object ID. If the context is wrong or object ID expires, it will fail to execute `__helios_seek`.

## Variations

### Variation A: Revert to Playwright evaluate for arguments
If raw `CDPSession` `callFunctionOn` is too cumbersome or fragile regarding object IDs, revert the main frame evaluation to use Playwright's built-in `evaluate` which handles argument passing correctly, just like the fallback frames do:
```typescript
        const promise = frame.evaluate(
          ([t, timeoutMs]) => (window as any).__helios_seek(t, timeoutMs),
          [timeInSeconds, this.timeout]
        );
```
