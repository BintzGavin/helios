#### 1. Context & Goal
- **Objective**: Implement robust argument parsing for `parseCommand` and perform infrastructure package housekeeping (version bump, add missing lint script).
- **Trigger**: The current `parseCommand` implementation naively splits by whitespace, breaking arguments that contain spaces inside quotes (e.g., `-metadata title="My Render"` or file paths with spaces). The `packages/infrastructure/package.json` is missing a `lint` script (causing `npm run lint` failures) and is incorrectly versioned `0.1.0` (it should be `0.12.0`).
- **Impact**: Enables workers to safely execute render commands containing strings with spaces. Ensures the infrastructure package is compliant with the repository's linting workflow.

#### 3. Implementation Spec
- **Architecture**:
  - The `parseCommand` function will use a regular expression or a state machine to extract arguments, preserving text inside single (`'`) and double (`"`) quotes. It should handle nested/escaped quotes if possible, but basic quote boundary matching is the minimum requirement.
  - The `package.json` version will jump from `0.1.0` directly to `0.12.0` to match the chronological `INFRASTRUCTURE.md` status logs.
  - The missing `lint` script will be added to `package.json` as `"lint": "tsc --noEmit"`.
- **Pseudo-Code**:
  ```typescript
  // In packages/infrastructure/src/utils/command.ts
  export function parseCommand(commandString: string): { command: string, args: string[] } {
    // 1. Trim input
    // 2. Use a regex to match:
    //    - Quoted strings: /"[^"]*"/ or /'[^']*'/
    //    - Unquoted strings: /[^\s]+/
    // 3. Extract matches, removing surrounding quotes from quoted arguments
    // 4. Return { command: matches[0], args: matches.slice(1) }
  }
  ```
- **Public API Changes**:
  - `parseCommand` behavior is modified, but its signature remains identical.
- **Dependencies**: None. This is a self-contained fix.
- **Cloud Considerations**: Accurate command parsing is critical for stateless worker entrypoints (`AwsLambdaAdapter`, `CloudRunAdapter`, etc.) as they pass these arguments directly to `child_process.spawn`.
