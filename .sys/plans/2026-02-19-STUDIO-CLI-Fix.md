# Context & Goal
- **Objective**: Fix the `helios studio` CLI command and package exports by aligning the build output filename.
- **Trigger**: The `vite.config.cli.ts` outputs `index.js` but `bin/helios-studio.js` and `package.json` expect `plugin.js`. This prevents `npx helios studio` from working and breaks programmatic usage.
- **Impact**: Enables the `helios studio` CLI command to function correctly and allows programmatic usage of the studio plugin via `import { studioApiPlugin } from '@helios-project/studio/cli'`.

# File Inventory
- **Modify**: `packages/studio/vite.config.cli.ts`
  - Change `fileName` from `'index'` to `'plugin'` to match `package.json` exports and bin script imports.
- **Read-Only**: `packages/studio/package.json`, `packages/studio/bin/helios-studio.js`

# Implementation Spec
- **Architecture**: Update Vite library mode configuration to output `plugin.js` instead of `index.js`.
- **Pseudo-Code**:
  ```typescript
  // packages/studio/vite.config.cli.ts
  export default defineConfig({
    build: {
      lib: {
        entry: path.resolve(__dirname, 'src/server/plugin.ts'),
        formats: ['es'],
        fileName: 'plugin' // Changed from 'index'
      },
      // ...
    }
  });
  ```
- **Public API Changes**: None (Fixes broken implementation).
- **Dependencies**: None.

# Test Plan
- **Verification**:
  1. Run `npm run build:cli` in `packages/studio`.
  2. Verify `packages/studio/dist/cli/plugin.js` exists (using `ls` or `fs.exists`).
  3. Attempt to run `node packages/studio/bin/helios-studio.js` (expecting it to proceed past module loading, even if it fails later due to missing build artifacts for UI).
- **Success Criteria**: Build produces `plugin.js` and the bin script can resolve the import.
- **Edge Cases**: Ensure `package.json` types exports (`plugin.d.ts`) also align (they point to `dist/types/server/plugin.d.ts` which is generated by `tsc` separately, so this change only affects the JS output, which is correct).
