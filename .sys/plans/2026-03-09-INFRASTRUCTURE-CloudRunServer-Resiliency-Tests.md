#### 1. Context & Goal
- **Objective**: Implement comprehensive resiliency and regression tests for `createCloudRunServer` error handling.
- **Trigger**: The Infrastructure agent has reached gravitational equilibrium with the V2 vision, and expanding error state coverage is a valid fallback action to ensure execution stability.
- **Impact**: Enhances the robustness of the Cloud Run entrypoint by verifying it behaves correctly under malformed payloads, non-POST requests, empty bodies, and internal execution errors, preventing unhandled promise rejections and silent failures in production.

#### 3. Implementation Spec
- **Architecture**: The tests will stand up the HTTP server generated by `createCloudRunServer` on a dynamic port and send various malformed HTTP requests to it to verify its resiliency. It will mock `WorkerRuntime` to simulate both successful executions and unexpected internal throws, verifying the server correctly maps these to appropriate HTTP status codes without crashing.
- **Pseudo-Code**:
  - Create a test block for `createCloudRunServer`.
  - Use `node:http` or a similar client to send requests to the server port.
  - Implement a test for a non-POST request (e.g., GET) and verify it returns a 405 Method Not Allowed.
  - Implement a test for an empty payload and verify it returns a 400 Bad Request.
  - Implement a test for a malformed JSON payload and verify it returns a 400 error (or gracefully handles the parse failure).
  - Implement a test for a payload missing `jobPath` or `chunkIndex` and verify it returns a 400 Bad Request.
  - Mock `WorkerRuntime.prototype.run` to return a successful execution (`{ exitCode: 0, stdout: 'ok', stderr: '' }`) and verify the server returns a 200 OK.
  - Mock `WorkerRuntime.prototype.run` to return a failed execution (`{ exitCode: 1, stdout: '', stderr: 'fail' }`) and verify the server returns a 500 status code with the stderr in the body.
  - Mock `WorkerRuntime.prototype.run` to throw an unexpected error (e.g., `new Error('Internal exception')`) and verify the server catches it and returns a 500 status code.
- **Public API Changes**: None.
- **Dependencies**: None.
- **Cloud Considerations**: The tests verify the specific contract expected by Google Cloud Run services (HTTP POST, standard status codes).

- **Verification**: `cd packages/infrastructure && npm run test -- tests/worker/cloudrun-server-resiliency.test.ts`
