# PERF-1053: Inline dispatches evaluation in CaptureLoop.ts worker assignment logic

## Focus Area
The worker assignment logic in `CaptureLoop.ts` (multi-worker writer chunk loops and the `checkState` single worker handler).

## Background Research
Currently, inside the multi-worker chunk dispatch logic (around lines 464, 588, and 653), the code evaluates `limit` and `dispatches` as:
```typescript
const limit = Math.min(nextFrameToWrite + maxPipelineDepth, totalFrames);
let dispatches = limit - nextFrameToSubmit;
```

This allocates an intermediate `limit` variable in the hot loop. By moving the calculation of `dispatches` into a single expression, we reduce the total number of variable declarations and slightly decrease AST size and dynamic evaluation overhead.
Since `limit` is only used to compute `dispatches`, we can inline it:
```typescript
let dispatches = Math.min(nextFrameToWrite + maxPipelineDepth, totalFrames) - nextFrameToSubmit;
```

This was previously planned in an earlier plan but not applied consistently across the multi-worker writer loops and the `checkState` worker logic. While micro-optimizations like this (e.g. PERF-947) were previously categorized as invalid for some local variable assignments (like `Math.min(maxSubmits)` where `maxSubmits = a+b`), in this case we are fully removing the variable allocation for `limit` and collapsing the expression in highly hot polling loops, similar to PERF-1045 which successfully inlined `maxSubmits`.
