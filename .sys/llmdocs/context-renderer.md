# Context: Renderer

## Section A: Strategy
The Renderer uses a "Dual-Path" architecture:
1. **Canvas Mode**: Captures frames by converting `<canvas>` to data URLs.
2. **DOM Mode**: Captures frames by taking full-page screenshots.

## Section C: Configuration
interface RendererOptions {
  width: number;
  height: number;
  fps: number;
  durationInSeconds: number;
  mode?: 'canvas' | 'dom';
}

## Section D: FFmpeg Interface
Input arguments are generated by the Strategy (e.g., piped input).
Output arguments:
- `-c:v libx264`
- `-pix_fmt yuv420p`
- `-movflags +faststart`
